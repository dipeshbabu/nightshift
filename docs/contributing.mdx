Contributions are welcome!

## Workflow

1. Fork and clone the repo
2. Install dependencies:
```bash
bun install
```
3. Make changes, write and run tests

```bash
bun test
```
4. Ensure build work:

```bash
bun run build:single
```

5. Open a PR 

## Getting started

If you are unsure where to start, or how to get involved join us in our [slack](https://join.slack.com/t/nightshiftoss/shared_invite/zt-3p5dshiiq-hjB8558QvURDgqqCI7e8RQ).

## Code style

- Use Bun APIs if you can  
- Follow existing patterns in the codebase
- Include tests for new functionality

## Pull requests

When opening a PR:

- Provide clear context about what the change does
- PRs need to come from issues that were discussed
- PRs need to reference the discussed issue
